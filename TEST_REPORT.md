## Тест-кейсы

### TC-01 — Регистрация нового пользователя (/start)

**Тип:** Unit (интеграция бот ↔ БД)  
**Приоритет:** Высокий

**Предусловия:** Пользователь ранее не запускал бота.

**Шаги:**
1. Открыть бот в Telegram.
2. Нажать кнопку **Start** или ввести команду `/start`.
3. Наблюдать ответ бота.

**Ожидаемый результат:**
- Бот отправляет приветственное сообщение с именем пользователя.
- Отображается главное меню с кнопками: «📄 Загрузить резюме», «🔍 Найти вакансии», «👤 Мой профиль», «❓ Помощь».
- В базе данных создаётся запись пользователя.

**Фактический результат:**
- Приветственное сообщение отображено корректно.
- Все кнопки главного меню присутствуют.
- Повторный `/start` не создаёт дублирующую запись (метод `get_or_create`).

**Статус:** ✅ PASS

### TC-02 — Загрузка резюме текстом

**Тип:** Integration  
**Приоритет:** Высокий

**Предусловия:** Пользователь прошёл TC-01.

**Шаги:**
1. Нажать «📄 Загрузить резюме».
2. Выбрать «Ввести текстом».
3. Отправить текст:
   ```
   Python разработчик, 3 года опыта.
   Навыки: Python, Django, PostgreSQL, Docker, Git, REST API.
   Образование: МГТУ им. Баумана, бакалавр, 2021.
   Желаемая должность: Backend Developer.
   ```
4. Наблюдать ответ бота.

**Ожидаемый результат:**
- Бот подтверждает принятие резюме.
- Отображаются извлечённые навыки: python, django, postgresql, docker, git.
- Указана желаемая должность: «Backend Developer».
- Резюме сохранено в БД (таблица `resumes`).

**Фактический результат:**
- Навыки извлечены корректно: `python, django, postgresql, docker, git, rest api`.
- Желаемая должность определена: «Backend Developer».
- Сообщение подтверждения получено.

**Статус:** ✅ PASS

### TC-04 — Поиск вакансий с фильтрами

**Тип:** Integration (бот ↔ TrudVsem API)  
**Приоритет:** Высокий

**Предусловия:** Загружено резюме (TC-02 или TC-03).

**Шаги:**
1. Нажать «🔍 Найти вакансии».
2. Ввести должность: `Python разработчик`.
3. Выбрать город: `Москва`.
4. Ввести зарплату: `120000`.
5. Выбрать опыт: `1–3 года`.
6. Выбрать занятость: `⏭ Пропустить`.
7. Дождаться результатов.

**Ожидаемый результат:**
- Отображается список до 5 вакансий с процентом совпадения.
- Каждая вакансия содержит: название, компанию, город, зарплату, % соответствия.
- Результаты отсортированы по убыванию релевантности.

**Фактический результат:**
- Найдено 5 вакансий, результаты отсортированы по score.
- Время поиска: ~20 секунд (загрузка NLP-модели + API-запрос).
- Процент соответствия: от 42% до 78%.

**Статус:** ✅ PASS

## Выводы

1. Основные пользовательские сценарии (регистрация, загрузка резюме, поиск, Skill Gap) работают корректно.
2. Валидация пользовательского ввода реализована: некорректная зарплата отклоняется без сбоя FSM.
3. NLP-парсинг резюме извлекает навыки с приемлемой точностью.
4. Время поиска (~20 с) обусловлено первичной загрузкой NLP-модели; при последующих запросах — ~5 с.

## Известные ограничения

- При первом запуске скачивается NLP-модель (~400 МБ) — занимает 2–5 мин.
- API TrudVsem может возвращать нестабильные результаты в зависимости от доступности.
